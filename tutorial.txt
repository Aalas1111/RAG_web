================================================================================
                    LightRAG Web 部署与调试教程
================================================================================

一、本地部署与测试
--------------------------------------------------------------------------------
1. 环境要求
   - Python 3.10 或以上
   - Node.js 18 或以上
   - 本地工作目录示例：D:\Coding\PycharmProjects\0 script\RAG_web

2. 配置 .env
   - 在项目根目录（RAG_web）下已有 .env 文件；若没有，可复制 .env.example 为 .env
   - 必须填写：
     DEEPSEEK_API_KEY=你的DeepSeek密钥
     SILICONCLOUD_API_KEY=你的硅基流动密钥
   - 可选修改：
     ADMIN_USERNAME=admin
     ADMIN_PASSWORD=admin123
     SECRET_KEY=生产环境请改为随机长字符串

3. 启动后端
   - 打开终端，进入后端目录并安装依赖、启动服务：
     cd "D:\Coding\PycharmProjects\0 script\RAG_web\backend"
     pip install -r requirements.txt
     uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
   - 看到 "Uvicorn running on http://0.0.0.0:8000" 即表示后端已启动
   - 可访问 http://127.0.0.1:8000/docs 查看 API 文档

4. 启动前端
   - 再开一个终端，进入前端目录并安装依赖、启动开发服务器：
     cd "D:\Coding\PycharmProjects\0 script\RAG_web\frontend"
     npm install
     npm run dev
   - 浏览器访问 http://localhost:5173
   - 前端会通过 Vite 代理将 /api 请求转发到 http://127.0.0.1:8000，无需额外配置

5. 本地测试流程建议
   - 先访问首页，此时“选择数据图谱”可能为空（尚未创建图谱）
   - 访问 http://localhost:5173/login 使用 .env 中的管理员账号登录
   - 登录后进入管理后台，上传一个 .txt 文件创建第一个图谱（填写名称、简介、每日限额）
   - 创建完成后回到首页，选择该图谱，选择查询模式（如 hybrid），输入问题并提交，查看下方 AI 回答
   - 在管理后台可查看“今日查询统计”、修改每日限额、对图谱做增量更新或删除


二、部署到服务器（CentOS 8 + 宝塔面板）
--------------------------------------------------------------------------------
1. 服务器与目录约定
   - 服务器 IP：8.136.107.55
   - 项目在服务器上的工作目录：/www/wwwroot/rag_web
   - 注意：宝塔面板下网站根目录常见为 /www/wwwroot/域名 或 /www/wwwroot/rag_web，请以实际为准

2. 上传代码与依赖
   - 将整个 RAG_web 项目上传到 /www/wwwroot/rag_web（可用 Git 或宝塔文件管理）
   - 不要上传 .env 中的真实密钥到公开仓库；在服务器上单独创建 .env
   - 在服务器上执行：
     cd /www/wwwroot/rag_web/backend
     pip3 install -r requirements.txt
     或使用宝塔“Python 项目”添加项目后在其虚拟环境中安装：
     pip install -r /www/wwwroot/rag_web/backend/requirements.txt
   - 前端需要先在本机或服务器上构建：
     cd /www/wwwroot/rag_web/frontend
     npm install
     npm run build
   - 构建完成后，前端静态文件在 frontend/dist 目录

3. 后端以服务方式运行（推荐 Supervisor 或宝塔 PM2）
   - 方式 A：宝塔“Python 项目”
     - 在宝塔中新建 Python 项目，选择项目路径为 /www/wwwroot/rag_web/backend
     - 启动方式选择 uvicorn，启动文件填 app.main:app，端口填 8000（或其它未被占用的端口）
     - 环境变量可从 .env 加载（若宝塔支持填写环境变量，可逐项填写）
   - 方式 B：Supervisor
     - 新建配置 /etc/supervisor/conf.d/rag_web.conf，示例：
       [program:rag_web_api]
       directory=/www/wwwroot/rag_web/backend
       command=/usr/bin/python3 -m uvicorn app.main:app --host 0.0.0.0 --port 8000
       environment=PATH="/www/server/pyenv/versions/你的Python路径/bin"
       autostart=true
       autorestart=true
       stdout_logfile=/www/wwwroot/rag_web/logs/uvicorn.log
       stderr_logfile=/www/wwwroot/rag_web/logs/uvicorn_err.log
     - 需先创建日志目录：mkdir -p /www/wwwroot/rag_web/logs
     - 加载配置：supervisorctl reread && supervisorctl update
   - 注意：CentOS 8 上若使用宝塔安装的 Python，可执行 which python3 或 which uvicorn 确认路径；宝塔默认路径可能与 /www/server 相关，请按实际路径修改

4. 前端静态资源与反向代理（Nginx）
   - 将 frontend/dist 内的文件部署到网站根目录，例如：
     - 网站根目录设为 /www/wwwroot/rag_web/frontend/dist
   - 或在 Nginx 中为该站点配置 root 指向该 dist 目录
   - 配置反向代理，将 /api 转发到后端（假设后端运行在 127.0.0.1:8000）：
     location /api {
         proxy_pass http://127.0.0.1:8000;
         proxy_set_header Host $host;
         proxy_set_header X-Real-IP $remote_addr;
         proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
         proxy_set_header X-Forwarded-Proto $scheme;
         proxy_connect_timeout 300s;
         proxy_send_timeout 300s;
         proxy_read_timeout 300s;
     }
   - 若前端使用 history 模式（Vue Router createWebHistory），需配置 try_files，否则刷新会 404：
     location / {
         try_files $uri $uri/ /index.html;
     }
   - 保存后重载 Nginx：nginx -t && nginx -s reload

5. 服务器上的 .env
   - 在 /www/wwwroot/rag_web 下创建 .env，内容与本地类似，填写服务器环境专用的 API Key 和管理员密码
   - 确保后端进程启动时能读到该目录下的 .env（例如工作目录为 /www/wwwroot/rag_web/backend 时，config 会从上级目录 PROJECT_ROOT 读 .env）

6. 数据与权限
   - 运行后会在项目根目录下生成 data/ 目录（SQLite 与各图谱数据），请保证运行后端进程的用户对该目录有写权限
   - 若使用宝塔，通常运行用户为 www 或 root，可 chown -R www:www /www/wwwroot/rag_web/data

7. 管理界面「重启网站」
   - 管理员在管理界面顶栏可点击「重启网站」，触发服务器上的重启脚本（前后端服务）
   - 需在 .env 或「配置环境变量」中设置 RESTART_SCRIPT，例如：RESTART_SCRIPT=scripts/restart.sh
   - 脚本路径为相对项目根目录；脚本必须在项目根目录下（安全限制）
   - 项目内提供示例 scripts/restart.sh：按 Supervisor 程序名 rag_web_api 重启后端，并可选重载 Nginx
   - 若使用宝塔「Python 项目」或其它方式，请复制 scripts/restart.sh 并修改其中的 supervisorctl/nginx 命令为你的实际重启方式（如 systemctl、宝塔重启接口等）
   - 未配置 RESTART_SCRIPT 时，点击「重启网站」会提示「未配置重启脚本」


三、各类调试方法
--------------------------------------------------------------------------------
1. 后端接口调试
   - 本地：浏览器访问 http://127.0.0.1:8000/docs 使用 Swagger 调试
   - 公开接口：GET /api/graphs（图谱列表）、POST /api/query（请求体：graph_id, query, mode）
   - 管理员接口：需在请求头加 Authorization: Bearer <token>，token 来自 POST /api/admin/login

2. 前端代理与跨域
   - 本地开发：前端 npm run dev 时，vite.config.js 已配置 proxy 将 /api 转到 8000 端口，一般无需改后端 CORS
   - 部署到服务器后：前端直接请求同域下的 /api，由 Nginx 转发到后端，不存在跨域问题
   - 若前端单独部署到其它域名，需在后端 main.py 中设置 allow_origins 为前端域名，或按需调整 CORS

3. 登录 401 或 token 失效
   - 检查 .env 中 ADMIN_USERNAME、ADMIN_PASSWORD 与登录时输入是否一致
   - 检查 SECRET_KEY 在重启后端后是否一致（改过 SECRET_KEY 会导致旧 token 失效）
   - 管理后台请求需带 Authorization: Bearer <token>，前端在 api.js 中从 localStorage 的 admin_token 读取

4. 查询报错 500 或超时
   - 查看后端日志（uvicorn 或 Supervisor 输出），确认是否为 LightRAG/DeepSeek/硅基流动 报错
   - 确认 .env 中 DEEPSEEK_API_KEY、SILICONCLOUD_API_KEY 正确且未过期
   - 大图谱或长文本首次构建/查询可能较慢，可适当调大 Nginx 与 uvicorn 的超时时间

5. 图谱列表为空或创建失败
   - 列表为空：先登录管理后台，上传 txt 创建至少一个图谱
   - 创建失败：查看后端日志；确认上传的是 .txt 且编码为 UTF-8 或 GBK；确认 data/ 目录可写
   - 若 LightRAG 报错，确认 lightrag-hku 已正确安装：pip show lightrag-hku

6. 今日查询次数与限额
   - 管理后台“今日查询统计”来自接口 GET /api/admin/stats
   - 修改限额后立即生效；超过限额后，用户端 POST /api/query 会返回 429，前端需展示友好提示

7. 宝塔面板路径差异
   - 宝塔默认 Python 可能在 /www/server/pyenv 或类似路径；网站根目录可能在 /www/wwwroot/你的域名
   - 若项目实际路径与本文不同，请将上述 /www/wwwroot/rag_web 全部替换为你的实际路径
   - Nginx 配置一般在宝塔“网站” -> “设置” -> “配置文件”中修改

8. 日志位置建议
   - 后端：uvicorn 或 Supervisor 配置中的 stdout_logfile / stderr_logfile
   - 可自行在 app/main.py 或各 router 中增加 logging，将日志写到 /www/wwwroot/rag_web/logs 便于排查


四、常用命令速查
--------------------------------------------------------------------------------
本地后端：  cd backend && uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
本地前端：  cd frontend && npm run dev
前端构建：  cd frontend && npm run build
查看 API 文档：  http://127.0.0.1:8000/docs
管理员登录：  前端访问 /login，或 POST /api/admin/login 获取 token

五、在 Cursor 内用 Git 提交到 GitHub
--------------------------------------------------------------------------------
1. 确认已安装 Git 与 Cursor 的 Git 相关功能（或 Git 插件）
   - 在 Cursor 左侧边栏可看到「源代码管理」图标（分支形状），或按 Ctrl+Shift+G 打开

2. 若项目尚未用 Git 管理
   - 打开「源代码管理」面板，若提示「未检测到 Git 存储库」，点击「初始化存储库」
   - 或在本机项目根目录打开终端，执行：git init

3. 确认 .gitignore
   - 项目根目录已有 .gitignore，会忽略 .env、data/、node_modules/、frontend/dist 等，避免把密钥和构建产物提交上去

4. 添加并提交
   - 在「源代码管理」中，将需要提交的文件勾选（或点「+」暂存全部变更）
   - 在顶部输入框填写提交说明（如「初始提交」或「添加重启网站功能」），点击 ✓ 提交
   - 或终端：git add . && git commit -m "你的提交说明"

5. 在 GitHub 上创建仓库
   - 登录 https://github.com ，右上角「+」->「New repository」
   - 仓库名自定（如 RAG_web），选 Public/Private，不要勾选「Add a README」（避免与本地冲突）
   - 创建后，页面上会显示「…or push an existing repository from the command line」

6. 关联远程并推送
   - 在 Cursor 终端执行（将 YOUR_USERNAME 和 RAG_web 换成你的用户名和仓库名）：
     git remote add origin https://github.com/YOUR_USERNAME/RAG_web.git
     git branch -M main
     git push -u origin main
   - 或使用 Cursor「源代码管理」：点击「发布分支」或「推送」前，先在设置里添加远程仓库地址
   - 若使用 SSH：git remote add origin git@github.com:YOUR_USERNAME/RAG_web.git

7. 之后日常使用
   - 修改代码后，在「源代码管理」中暂存、写提交说明、提交；再点「同步」或「推送」到 GitHub
   - 或终端：git add . && git commit -m "说明" && git push
